# 07장 - 연산자

## 이번 장 핵심

- 연산자는 하나 이상의 표현식을 대상으로 값을 만든다.
- 동등 비교(`==`)는 암묵적 타입 변환을 수행한다.
- 일치 비교(`===`)는 타입과 값이 모두 같아야 한다.
- 산술 연산 과정에서도 암묵적 타입 변환이 일어날 수 있다.

## 헷갈렸던 점

- `1 == "1"`은 왜 `true`일까?
- `"1" + 1`과 `"1" - 1`은 왜 다르게 동작할까?
- `true + 1`은 왜 숫자처럼 계산될까?

## 실험 코드

- `operators.js` - 07장 실험 목록
- `equality.js` - 동등 비교와 일치 비교 확인
- `implicit-conversion.js` - 연산 중 암묵적 타입 변환 확인

## 실행 방법

```bash
node 07-operators/operators.js
node 07-operators/equality.js
node 07-operators/implicit-conversion.js
```

## 실행 결과

```txt
1 == "1": true
1 === "1": false
"1" + 1: 11
"1" - 1: 0
true + 1: 2
```

## 알게 된 점

- `==`는 타입 변환 때문에 예상보다 느슨하게 비교된다.
- `===`는 타입까지 비교하므로 백엔드 검증 코드에서 더 안전하다.
- `+`는 문자열 연결과 숫자 덧셈에 모두 쓰여 헷갈릴 수 있다.
- `-`는 문자열 연결 용도가 없어서 숫자 변환을 시도한다.

## Node.js 백엔드와 연결

- query string으로 받은 값은 문자열이므로 `req.query.page === 1`은 보통 `false`다.
- 페이지 번호, limit, difficulty 같은 값은 명시적으로 숫자로 변환해야 한다.
- API validation에서는 암묵적 타입 변환에 기대지 않는 편이 안전하다.
